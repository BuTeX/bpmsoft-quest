# План закрытого пилота

## Цель

Проверить, что тренажёр помогает аналитикам принимать и объяснять решения, а не только угадывать ответы, и что production-контур выдерживает реальное прохождение без потери прогресса.

## Состав

- 12–20 участников;
- минимум 5 начинающих аналитиков, 5 аналитиков с опытом и 2 наставника/эксперта BPMSoft;
- один владелец пилота, один контент-эксперт и один технический дежурный;
- длительность — 10 рабочих дней.

## Сценарий

1. День 0: проверить согласия, создать backup, раздать доступ и канал поддержки.
2. Дни 1–3: Академия и «Медные машины».
3. Дни 4–7: «Семь дорог» и «Золотая полка».
4. Дни 8–9: «Гуд Авиа», экспорт данных и повторное прохождение одного задания.
5. День 10: интервью на 20 минут и решение о следующей стадии.

## Метрики принятия

| Метрика | Порог пилота |
|---|---:|
| Участники, успешно вошедшие и сохранившие прогресс | ≥ 95% |
| Завершившие минимум 9 заданий | ≥ 80% |
| Завершившие все 45 заданий | ≥ 40% |
| Медиана ошибок на проверку после первой карты | снижается минимум на 20% |
| Потеря или откат подтверждённого прогресса | 0 случаев |
| Необъяснимые ответы/разборы по оценке SME | 0 критических, ≤ 3 некритических |
| Критические accessibility-блокеры | 0 |
| p95 API в пилотной нагрузке | < 1 сек |
| Ошибки 5xx | < 1% |

Показатели обучения сравниваются по реальным `answer_checked`, `mission_started` и `mission_completed`, а не по снимку текущего прогресса.

## Интервью

1. В каком задании цель заказчика была непонятна?
2. Где правильный ответ удалось угадать без понимания?
3. Какой разбор помог исправить причинную ошибку?
4. Где интерфейс заставил искать следующий шаг?
5. Можно ли после задания объяснить решение коллеге без подсказки?
6. Что помешало продолжить на другом устройстве?
7. Какие данные участник не ожидал увидеть в экспорте?

## Правила остановки

Пилот немедленно приостанавливается при утечке данных, повторяемой потере прогресса, обходе доступа, неверном критическом знании BPMSoft или недоступности удаления аккаунта. Возобновление — только после исправления, регресса и подтверждения владельца пилота.

## Выход

Решение «расширять / повторить пилот / остановить» принимает владелец по таблице метрик, журналу инцидентов и подписанным заключениям SME, legal/privacy и accessibility-reviewer.
