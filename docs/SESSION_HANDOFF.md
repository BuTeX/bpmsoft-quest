- Кодовые названия, секретные операции и абстрактные магические причины не используются.
- Текстовая правка не меняет правильный ответ, механику, XP и порядок открытия.

## 5. Технические инварианты

- состояния карт хранятся раздельно;
- сервер канонизирует непрерывный completion-rank;
- режим изучения не записывает основной прогресс;
- варианты перемешиваются, а правильный не остаётся первым;
- повторное прохождение не выдаёт XP повторно;
- неизвестные поля состояния не принимаются сервером;
- имена файлов ассетов и внутренние ключи можно менять только с миграцией и тестами.

Ключи:

```text
bpmsoft-quest-v1
bpmsoft-quest-chapter2-v1
bpmsoft-quest-chapter3-v1
bpmsoft-quest-chapter4-v1
```
