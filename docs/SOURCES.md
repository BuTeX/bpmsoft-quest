## Как работать с источниками для миссии 08

Для каждого из 12 требований:

1. найти подтверждение базового функционала или настраиваемого механизма;
2. записать точный файл-источник в содержательную спецификацию;
3. не считать функцию коробочной только по общему знанию CRM;
4. отделять low-code настройку от интеграции;
5. назначать «Разработка» только при подтверждённом отсутствии коробочного/low-code пути;
6. назначать `TBD`, если классификация зависит от лицензии, объёма, нефункциональных требований или неизвестного внешнего контракта.

## Как работать с источниками для миссии 09

Сборка мини-решения разделена на шесть самостоятельных узлов: раздел/карточка, справочник, роли и права, DCM-кейс, BPMN-процесс уведомления и дашборд. Проверять не только наличие каждого механизма, но и его границу ответственности: поле «Ответственный» не заменяет права по записям, статус не заменяет DCM-кейс, ручное письмо не заменяет автоматизацию, а печатный отчёт — оперативный дашборд.

## Как работать с источниками для миссий 10–18

Подробные ответы и ссылки зафиксированы в [`CHAPTER_2_MISSIONS.md`](CHAPTER_2_MISSIONS.md).

Критичные границы:

- миссия 10: несколько полей поиска дубля объединяются через И; пустые ячейки не очищают заполненные поля при обновлении;
- миссия 11: портал не заменяет права по записям и колонкам; новые правила для существующих записей требуют актуализации;
- миссия 12: Basic применяется через HTTPS, пароль хранится как «Зашифрованная строка», значения секретов вводятся на целевой среде повторно;
- миссия 13: выполняется актуальная версия процесса; журналирование и сериализация решают разные задачи;
- миссия 14: поставочные пакеты не изменяются напрямую, циклические зависимости запрещены, нужные данные привязываются явно;
- миссия 15: журнал изменений отвечает за бизнес-поля, журнал аудита — за системные события; прямой SQL может пройти мимо журнала изменений;
- миссия 16: проблема, изменение и релиз — разные сущности; отдельную коробочную сущность CAB не придумывать;
- миссия 17: до обучения проверяются задача, сервис, лицензия и качество данных; ML/LLM не подменяют права и обязательные проверки;
- миссия 18: каждое исправление должно опираться на наблюдаемый симптом и проверяемый источник, а не на общий совет.
