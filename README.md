# Академия Гуд программ — BPMSoft Quest

BPMSoft Quest — практический веб-тренажёр по функционалу и low-code возможностям BPMSoft. Игрок заканчивает Академию, выходит на работу аналитиком и проходит три коммерческих проекта в фэнтезийном мире с узнаваемыми компаниями и рабочими ситуациями.

Полный статус проекта находится в [`PROJECT_STATUS.md`](PROJECT_STATUS.md).

## Карты и история

1. **Академия аналитиков** — основы интерфейса, данных, прав, процессов, кейсов, интеграций, аналитики и проектирования решения.
2. **АО «Медные машины»** — первый проект в компании «Системная практика» для производителя големов и комплектующих.
3. **Группа «Семь дорог»** — развитие внедрённой CRM и внутреннего центра компетенций транспортно-складской компании.
4. **Сеть «Золотая полка»** — омниканальная трансформация объединённой торговой сети.

Фэнтези используется как фон и отраслевой колорит. Все задания начинаются с роли заказчика, бизнес-проблемы и проверяемых фактов. Канон и правила формулировок закреплены в [`docs/CONTENT_AND_MISSION_RULES.md`](docs/CONTENT_AND_MISSION_RULES.md).

## Запуск

```bash
npm install
npm start
```

Приложение будет доступно на `http://localhost:4173`.

С `DATABASE_URL` аккаунты, серверные сессии и прогресс четырёх карт сохраняются в PostgreSQL. Без `DATABASE_URL` сервер использует временное хранилище в памяти, поэтому данные исчезают после перезапуска. Переменные окружения перечислены в [`.env.example`](.env.example), схема таблиц — в [`db/schema.sql`](db/schema.sql).

Для запуска в локальной сети:

```bash
./start-lan.sh
```

## Что реализовано

- 36 заданий на четырёх отдельных картах;
- последовательное прохождение и свободный режим изучения;
- аккаунт по email и паролю, `scrypt`, `HttpOnly`/`SameSite` cookie и 30-дневная сессия;
- серверная синхронизация четырёх сохранений и локальный кэш в `localStorage`;
- XP, попытки, достижения, повторное прохождение без повторной награды;
- разные интерактивные механики для последовательностей, сопоставлений, прав, процессов, классификации, проектирования и многоэтапной диагностики;
- отдельные прологи, карты и финалы всех четырёх глав;
- новая механика четвёртой карты: исследование хотспотов и сборка цепочки «причина → механизм → проверка»;
- подсказки, разбор ошибок, частичная фиксация верных ответов и админ-режим;
- адаптивный интерфейс, управление с клавиатуры и `prefers-reduced-motion`;
- автоматические тесты маршрутов 01–36, состояний, аккаунтов и серверной раздачи.

## Граница контента

Текстовая переработка не меняет механику, правильные ответы, порядок открытия, XP, ключи сохранения и подтверждённые сведения о BPMSoft. Новая функция платформы или другой правильный ответ требуют отдельного продуктового изменения, источника и теста.

## Фирменный стиль и арты

- официальный логотип используется без перерисовки и эффектов;
- основная гарнитура — локальный `Unbounded` Light/Regular/Bold;
- базовая палитра: `#1B0B3B`, `#7856FF`, `#FE5500`, `#00C1D4`, `#FFA51E`, `#EDE8F5`;
- имена файлов и промпты ассетов зафиксированы в [`assets/image-prompts.md`](assets/image-prompts.md).

Технические имена существующих файлов ассетов сохраняются для совместимости, даже если видимые названия заданий изменились.
