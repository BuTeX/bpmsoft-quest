Игровое веб-приложение Академии Гуд программ для изучения функционала и low-code возможностей BPMSoft.

## Запуск

Для полного режима с сервером установите зависимости и запустите приложение:

```bash
npm install
npm start
```

Приложение откроется на `http://localhost:4173`. Без `DATABASE_URL` оно продолжит работать с локальным сохранением; с PostgreSQL прогресс синхронизируется между перезагрузками через сервер.

Переменные окружения перечислены в [`.env.example`](.env.example). Таблица `player_progress` создаётся автоматически при запуске по схеме [`db/schema.sql`](db/schema.sql).

## Доступ из локальной сети

Запустите из корня проекта:

```bash
./start-lan.sh
```

Скрипт поднимает HTTP-сервер на `0.0.0.0:4173` и выводит LAN-ссылки компьютера. Другие устройства должны быть подключены к тому же роутеру.

## Что работает

- карта мира с зонами платформы;
- отдельное двухабзацное фэнтези-вступление при первом входе в каждое задание;
- профиль игрока, XP и low-code energy;
- уровень 1 `Основы платформы`: пять залов от интерфейса и данных до процессов и DCM;
- уровень 2 `Архитектура решения`: проектирование составных low-code решений;
- задание 01 `Interface Hub`: цепочка «рабочее место → раздел → реестр → карточка → деталь»;
- задание 02 `Data Forge`: сопоставление терминов «объект, поле, справочник, деталь, связь» с определениями;
- задание 03 `Access Citadel`: пять последовательных вопросов о ролях и уровнях прав доступа;
- задание 04 `Process Engine`: сборка low-code процесса обработки входящего email;
- задание 05 `Case Arena`: настройка элементов DCM-кейса непосредственно в пяти точках арены;
- задание 06 `Integration Harbor`: сборка пяти причалов входящего webhook-шлюза на панораме и проверка нового, повторного и неавторизованного запросов;
- задание 07 `Insight Tower`: последовательный диалог с проводником о выборе показателя, графика, списка, сводной таблицы и шкалы;
- задание 08 `Solution Gate`: классификация 12 требований по пяти повторно используемым категориям — коробка, low-code, интеграция, разработка и TBD;
- задание 09 `City Nexus`: сборка шести узлов мини-решения и приёмка регистрации, доступа и эскалации;
- финальный экран первого города с итогами `9 / 9`, `500 XP`, достижением и переходом к будущей отдельной карте;
- проверка, ловушки, подсказки, повторные попытки и отдельный прогресс уровня 2;
- сохранение прогресса в `localStorage` и синхронизация с PostgreSQL при серверном запуске;
- отдельные PNG-панорамы миссий, карта мира и портрет проводника.

## Арты

Промпты в палитре Гуд программ и точные имена файлов находятся в `assets/image-prompts.md`.

## Фирменный стиль

- официальный логотип используется из исходного PNG без изменений;
- основная гарнитура `Unbounded` подключена локально в начертаниях Light, Regular и Bold;
- интерфейс использует цвета брендбука `#1B0B3B`, `#7856FF`, `#FE5500`, `#00C1D4`, `#FFA51E`, `#EDE8F5`;
- логотип размещается только на светлом однотонном фоне с охранным полем;
- эффекты, деформации и перерисовка логотипа не используются.

## Граница текущей версии

Первый город полностью завершён: девять миссий, два уровня и суммарные `500 XP`. Следующая глава будет отдельной картой в другой стилистике с усложнёнными заданиями; она пока не спроектирована и не смешивается с прогрессией текущего города.
