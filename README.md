# Академия Гуд программ — BPMSoft Quest

BPMSoft Quest — практический веб-тренажёр по функционалу и low-code возможностям BPMSoft. Игрок заканчивает Академию, выходит на работу аналитиком и проходит четыре коммерческих проекта в фэнтезийном мире с узнаваемыми компаниями и рабочими ситуациями.

Полный статус проекта находится в [`PROJECT_STATUS.md`](PROJECT_STATUS.md).

## Карты и история

После входа игрок попадает на единую карту мира: она показывает весь карьерный маршрут, суммарный прогресс и доступность глав. Из неё можно открыть любую доступную локальную карту, а затем вернуться обратно без потери состояния.

1. **Академия аналитиков** — основы интерфейса, данных, прав, процессов, кейсов, интеграций, аналитики и проектирования решения.
2. **АО «Медные машины»** — первый проект в компании «Системная практика» для производителя големов и комплектующих.
3. **Группа «Семь дорог»** — развитие внедрённой CRM и внутреннего центра компетенций транспортно-складской компании.
4. **Сеть «Золотая полка»** — омниканальная трансформация объединённой торговой сети.
5. **Авиакомпания «Гуд Авиа»** — устойчивость пассажирских и операционных процессов в облачном мегахабе.

Фэнтези используется как фон и отраслевой колорит. Все задания начинаются с роли заказчика, бизнес-проблемы и проверяемых фактов. Канон и правила формулировок закреплены в [`docs/CONTENT_AND_MISSION_RULES.md`](docs/CONTENT_AND_MISSION_RULES.md).

## Запуск

```bash
npm install
npm start
```

Приложение будет доступно на `http://localhost:4173`.

- `http://localhost:4173/academy.html` — основной интерфейс Академии;
- `http://localhost:4173/update` — совместимый алиас на тот же интерфейс.

С `DATABASE_URL` аккаунты, серверные сессии, события и прогресс пяти карт сохраняются в PostgreSQL. При старте сервер последовательно применяет версионированные миграции из [`db/migrations`](db/migrations); отдельно их можно запустить командой `npm run db:migrate`. [`db/schema.sql`](db/schema.sql) остаётся актуальным снимком схемы для ревью. Без `DATABASE_URL` сервер использует временное хранилище в памяти, поэтому production-запуск должен задавать `REQUIRE_DATABASE=true`. Переменные окружения перечислены в [`.env.example`](.env.example).

Для запуска в локальной сети:

```bash
./start-lan.sh
```

## Что реализовано

- единая карта мира с пятью регионами, маршрутом карьеры, общей шкалой `0–3100 XP` и переходами к локальным картам;
- 45 заданий на пяти отдельных картах;
- последовательное прохождение и свободный режим изучения;
- аккаунт по email и паролю, `scrypt`, `HttpOnly`/`SameSite` cookie и 30-дневная сессия;
- серверная синхронизация пяти сохранений и локальный кэш в `localStorage`;
- XP, попытки, достижения, повторное прохождение без повторной награды;
- разные интерактивные механики для последовательностей, сопоставлений, прав, процессов, классификации, проектирования и многоэтапной диагностики;
- отдельные прологи, карты и финалы всех пяти глав;
- механика четвёртой карты: исследование хотспотов и последовательный выбор звеньев «причина → решение → проверка» из двух вариантов;
- новая механика пятой карты: детерминированный операционный двойник с временной шкалой, поиском первого расхождения, изменением правил и контрольным прогоном;
- подсказки, разбор ошибок, частичная фиксация верных ответов и админ-режим;
- отдельная админ-аналитика на `admin.html`: 5 вкладок, 30 дашбордов, фильтры и детализация по 45 квестам;
- адаптивный интерфейс, управление с клавиатуры и `prefers-reduced-motion`;
- автоматические тесты маршрутов 01–45, состояний, аккаунтов и серверной раздачи.

## Граница контента

Текстовая переработка не меняет механику, правильные ответы, порядок открытия, XP, ключи сохранения и подтверждённые сведения о BPMSoft. Новая функция платформы или другой правильный ответ требуют отдельного продуктового изменения, источника и теста.

## Фирменный стиль и арты

- официальный логотип используется без перерисовки и эффектов;
- основная гарнитура — локальный `Unbounded` Light/Regular/Bold;
- базовая палитра: `#1B0B3B`, `#7856FF`, `#FE5500`, `#00C1D4`, `#FFA51E`, `#EDE8F5`;
- имена файлов и промпты ассетов зафиксированы в [`assets/image-prompts.md`](assets/image-prompts.md).

Технические имена существующих файлов ассетов сохраняются для совместимости, даже если видимые названия заданий изменились.

## Текущее состояние

Все пять карт реализованы. Добавлены событийная аналитика, конфликтобезопасная синхронизация, миграции PostgreSQL, полный жизненный цикл аккаунта, публичный лендинг, CI/E2E и эксплуатационный контур. Технический статус — **готов к закрытому пилоту**. Публичный запуск заблокирован до прохождения юридической, предметной, accessibility- и пилотной приёмки из [`docs/LAUNCH_CHECKLIST.md`](docs/LAUNCH_CHECKLIST.md).
